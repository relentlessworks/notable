Quick Start

# Build the single static binary make build # Run with zero config — starts on :8080 ./notable # Create a note curl -X POST http://localhost:8080/api/notes \ -H "Authorization: Bearer <token>" \ -d "title=Meeting Notes&body=Discussed API design&tags=work,meeting" # Response (plain text, one line per record) handle=note_a1b2c title=Meeting Notes tags=work,meeting updated=2024-01-01T00:00:00Z # Search notes curl 'http://localhost:8080/api/notes/search?q=API' \ -H "Authorization: Bearer <token>" # → handle=note_a1b2c title=Meeting Notes tags=work,meeting updated=...

Principles

🤖

Agent IS the Interface

No UI, no SDK. The AI agent is the user. The API is the product.

📝

Plain Text by Default

One labeled, grepable line per record. Token-cheap, survives context truncation. JSON on demand.

💡

Instructive Errors

Every 4xx includes a hint telling the agent what to do next. Self-correcting without a schema.

📖

Self-Documenting

GET /help returns a one-page operating manual the agent loads at runtime.

🔐

Simple Auth

OTP via email → long-lived bearer token. No complex OAuth flows.

📦

Single Static Binary

Go, zero external dependencies, CGO_ENABLED=0. Deploys as one file.

⚙️

Zero Config Defaults

Runs out of the box. Config layered: defaults < env < flags.

🏢

Multi-Tenant Ready

Workspaces isolate notes per tenant. Plans, limits, audit logs built in.

API Reference

Authentication

POST /auth/request email=<email>&workspace=<handle> → OTP code POST /auth/verify email=<email>&code=<code> → Bearer token

Notes (requires Bearer token)

MethodPathDescription
POST/api/notesCreate a note
GET/api/notesList notes (optional ?tag=)
GET/api/notes/<handle>Get full note with body
PATCH/api/notes/<handle>Update a note
DELETE/api/notes/<handle>Delete a note
GET/api/notes/search?q=Search notes by content
GET/api/workspaceWorkspace info

Response Formats

# Plain text (default) handle=note_a1b2c title=Meeting Notes tags=work,meeting updated=2024-01-01T00:00:00Z # JSON (add Accept: application/json or ?format=json) {"handle":"note_a1b2c","title":"Meeting Notes","body":"Discussed API design","tags":["work","meeting"]}
